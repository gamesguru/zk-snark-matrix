% ── Abstract ──────────────────────────────────────────────────────
\begin{abstract}
We present a graph-native STARK framework designed for the
end-to-end verifiable reduction of Directed Acyclic Graphs (DAGs).
Traditional zero-knowledge virtual machines (zkVMs) emulate Von
Neumann architectures, incurring prohibitive memory permutation
overheads across millions of emulated CPU cycles~($C$).  Our
framework bypasses CPU emulation entirely, replacing temporal
instruction cycles with a static, spatial Bene\v{s} routing
network.  Furthermore, we identify and eliminate the ``trust
boundary seam'' prevalent in heterogeneous coprocessor models,
where data extraction and signature verification occur off-circuit.
Leveraging the arithmetic compatibility of Post-Quantum
Cryptography (PQC) with characteristic-2 fields, our framework
natively arithmetizes content hashing and signature verification
directly into the $\mathbb{F}_2$ execution trace.  The architecture
utilizes an Expander-based Locally Testable Code (LTC) for trace
commitment, requiring zero prime-field Fast Fourier Transforms
(FFTs).  To eliminate monolithic hardware exhaustion, the framework
integrates recursive SNARK composition, enabling horizontal
MapReduce proving and Incrementally Verifiable Computation (IVC).
The verifier confirms the absolute cryptographic authenticity and
topological consistency of the network state in $\approx 5$\,ms,
demonstrating a practical paradigm for trustless decentralized
consensus.
\end{abstract}


% ==================================================================
\section{Introduction}\label{sec:intro}

The demand for verifiable computation has driven rapid advancements
in Scalable Transparent Arguments of Knowledge (STARKs).  The
prevailing industry trajectory favors general-purpose
zkVMs~\cite{sp1,jolt}, which compile high-level code into RISC-V
or WebAssembly instruction traces and prove their correct execution
cycle-by-cycle.  While offering unmatched developer ergonomics,
zkVMs introduce a catastrophic computational bottleneck for
applications processing highly structured, large-scale data---such
as deterministically reducing concurrent states in blockchains,
build systems, and federated DAGs.

The bottleneck stems fundamentally from the Von Neumann
architecture.  Proving that a specific value was correctly written
and later read from RAM requires a global memory consistency check,
forcing the prover to compute expensive algebraic fractions over
large prime fields.  Furthermore, verifying cryptographic identity
(e.g., asymmetric signatures) inside these virtual machines triggers
massive non-native arithmetic penalties.  The standard industry
workaround is the heterogeneous coprocessor model: executing
signatures off-circuit, generating a secondary proof, and weakly
binding the proofs via cryptographic hashes.  This introduces a
critical trust boundary seam allowing malicious provers to exploit
data-extraction ambiguities.

Our framework abandons both the virtual machine and the coprocessor
seam.  Rather than emulating a topological walk cycle-by-cycle
inside an Instruction Set Architecture (ISA), we embed the DAG
directly into the STARK execution trace as a static spatial routing
network.  By capitalizing on the native $\mathbb{F}_2$ efficiency
of emerging PQC standards, we introduce a unified, single-layer
binary STARK that structurally binds identity, content hashing, and
topological reduction into a mathematically unassailable execution
trace.

\section{The Architecture of Trust Boundaries}\label{sec:bottleneck}

To formalize the asymptotic and security advantages of our unified
graph-native framework, we must deconstruct both the runtime
composition of standard zkVMs and the cryptographic fragility of
off-circuit data ingestion.

\subsection{The Asymptotic Cost of ISA Emulation}

General-purpose zkVMs operate by emulating a CPU\@.  Let $N$ be the
number of nodes in a DAG, and $C$ be the total CPU cycles required
to process that DAG natively.  To prove RAM consistency, trace-based
zkVMs rely on global permutation arguments (e.g., LogUp).  Every
memory access generates a fractional algebraic term
$\frac{1}{X - \mathrm{val}}$.  For a standard topological sort
algorithm, manipulating pointers and dynamically allocating arrays
in RISC-V inflates the cycle count $C$ to orders of magnitude larger
than $N$.  Thus, $C = K_{emu} \cdot N$, where the emulation constant
$K_{emu}$ typically ranges from $10^3$ to $10^4$.

The prover is bound by $\mathcal{O}(C \log C)$ trace operations
over a massive prime field to compute these rational functions.
While sum-check protocols over lookups
(e.g., Jolt~\cite{jolt}) optimize the
cryptographic operations to $\mathcal{O}(C)$, the underlying trace
remains bound to the cycle counter.  The constant $K_{emu}$ is
overwhelmingly dominated by instruction decoding and register
allocation overhead, rendering the arithmetic circuit profoundly
inefficient for strict topological reductions.

\begin{remark}[Asymptotic Crossover]\label{rem:crossover}
Constant-degree superconcentrator-based Expander
graphs~\cite{valiant} achieve a theoretical $\mathcal{O}(N)$ routing
cost, but the hidden constant is enormous: explicit constructions
carry combinatorial overhead so large that the crossover with
$\mathcal{O}(N \log_2 N)$ Bene\v{s}
routing~\cite{benes} occurs only at input sizes on the order of
$N \gtrsim 2^{75}$.  Even at $N = 2^{20}$
(${\approx}\,10^6$ nodes), a Jolt prover processes
${\sim}\,10^{9}$--$10^{10}$ emulated cycles while our framework
processes ${\sim}\,2 \times 10^{7}$ routing operations---a gap of
two to three orders of magnitude.
\end{remark}

\subsection{The Heterogeneous Coprocessor Seam}

Because standard signature schemes like Ed25519 operate over large
prime fields (e.g., Curve25519 where $p = 2^{255} - 19$), computing
them inside a binary field ($\mathbb{F}_2$) requires hundreds of
thousands of non-native constraints per operation.  To avoid this,
protocols often decouple identity from logic.  A native host parses
the payload, verifies the signature off-circuit, hashes the
extracted fields into a public input root, and feeds the values into
the STARK logic circuit.

This creates a critical cryptographic seam.  The proof attests that
the logic was executed correctly over the provided inputs, but it
cannot mathematically guarantee that those inputs were faithfully
extracted from the originally signed payload.  A malicious prover
can verify a genuine signature over a payload containing variable~$X$,
but inject variable~$Y$ into the STARK logic trace while maintaining
valid boundary hashes.  Closing this seam requires structurally
binding the cryptographic identity strictly to the exact witness
wires consumed by the logic circuit.

Our framework resolves both the Von Neumann memory bottleneck and
the coprocessor trust seam by deploying a single-layer,
characteristic-2 STARK\@.  We project the topological dependencies
directly onto a static Bene\v{s} permutation network and push
payload hashing and signature verification inside the trace boundary.

\subsection{Trace Embedding and Spatial Memory}

The framework initialization allocates a static $\mathbb{F}_2$
execution trace of width~$W$ (padded to $2^d \ge N$) and depth~$D$.
The input layer ingests the raw, unparsed payload bytes.  Rather
than executing pointer arithmetic, the intermediate layers function
as a physical Bene\v{s} network.  Guided by private plaintext
routing hints computed by the prover, the network recursively copies
required state variables downward and across adjacent column edges
using native bitwise XOR operations.  If a trace requires $N$
inputs, the Bene\v{s} network requires exactly
$2 \log_2(N) - 1$ layers, yielding a strict spatial complexity of
$\mathcal{O}(N \log N)$ trace nodes.  Each switch enforces a single
local algebraic constraint over $\mathbb{F}_2$:
\begin{equation}\label{eq:switch}
  \texttt{out} = a \oplus s \cdot (a \oplus b),
\end{equation}
where $s \in \{0,1\}$ is the private switch state.  Because memory
routing is enforced via geometric adjacency rather than fractional
polynomials, the constraint degree remains minimal.

\subsection{In-Circuit PQC and Content Hashing}

To eliminate the trust seam, the binary execution trace asserts the
mathematical validity of the raw input bytes before allowing them to
route through the network.  This is facilitated by the arithmetic
synergy of NIST Post-Quantum Cryptography (PQC) standards.

While classical elliptic curve cryptography requires prime-field
arithmetic, PQC algorithms map elegantly to binary fields.  The
SLH-DSA signature scheme (FIPS~205) is entirely hash-based, relying
exclusively on Keccak permutations comprising native bitwise AND,
XOR, and rotation operations.  Alternatively, lattice-based schemes
like FN-DSA (FIPS~206, Falcon) utilize Number Theoretic Transforms
(NTTs) over a highly constrained 14-bit prime ring ($q=12289$).
While this requires modulus logic, 14-bit integer multiplications
are efficiently executed via schoolbook multiplication inside
$\mathbb{F}_2$, yielding a tractable $\approx 5 \times 10^6$
constraints per signature.

By asserting $\texttt{Hash}(\text{canonical\_payload})
= \text{identity\_hash}$ alongside the PQC signature verification
directly in the upper layers of the trace, the exact bit-wires
representing the validated payload content flow continuously into
the logic comparators.  Equivocation between the identity layer and
the logic layer is rendered topologically impossible.

\subsection{Expander-Based Commitment (LTC)}

Our framework replaces algebraic FFT commitments with proximity
testing over a sparse Cryptographic Graph.  The prover stretches the
binary grid by multiplying it against a public, sparse,
constant-degree Expander matrix~$G$:
\[
  T_{\mathrm{stretched}} \;=\; T_{\mathrm{original}} \cdot G
\]
Because $G$ possesses a strict constant degree, matrix multiplication
over $\mathbb{F}_2$ is strictly linear relative to the trace size.
By the Expander mixing lemma, any localized constraint
failure---such as a single falsified Bene\v{s} switch
flag---cascades to corrupt a $\ge \delta$-fraction (e.g., $10\%$)
of the extended grid.  The prover Merkle-commits these stretched
columns to produce the trace commitment root.

\section{Recursive MapReduce Proving}\label{sec:recursive}

While the graph-native trace achieves optimal asymptotic complexity,
generating a monolithic proof for an infinitely expanding DAG
eventually exhausts hardware memory limitations.  Furthermore,
re-proving an entire causal history to append a single node is
temporally unviable.  The framework solves this via recursive SNARK
composition, enabling horizontal parallelism and Incrementally
Verifiable Computation (IVC).

\subsection{Horizontal Graph Partitioning}

Directed Acyclic Graphs naturally consist of disjoint causal branches
that ultimately converge.  This topology maps perfectly to a
MapReduce processing architecture.  A massive DAG is partitioned
into independent sub-graphs $\mathcal{D}_1, \dots, \mathcal{D}_m$.
Disparate servers within the network concurrently generate proofs
$\pi_1, \dots, \pi_m$ for their assigned sub-graphs.

A recursive circuit subsequently ingests these sub-proofs as private
witnesses, verifies their execution logic within the trace, and
outputs a singular aggregated proof $\pi_{agg}$.  This recursive
folding distributes the worst-case $\mathcal{O}(N^2)$ reduction
burden across commodity hardware, bounding wall-clock processing
time strictly to the depth of the recursive tree.

\subsection{Incrementally Verifiable Computation}

To support perpetually growing datasets, the framework employs IVC\@.
When new nodes append to the verified DAG, the network does not
reconstruct the entire causal history.  Instead, the prover
generates a sequential epoch proof $\pi_{i+1}$ that asserts:
$\texttt{Verify}(\pi_i) \wedge \texttt{Verify(DAG\_Delta)}$.  This
recursive fold confines the STARK trace complexity strictly to the
localized delta of newly appended events, bounding the sequential
runtime to $\mathcal{O}(\Delta N \log \Delta N)$ per epoch and
guaranteeing $\mathcal{O}(1)$ temporal verification for end clients
regardless of the network's age.

\section{Formalization of Cryptographic Boundaries}\label{sec:theory}

\begin{theorem}[Non-Blocking Trace Embeddability]\label{thm:embed}
Let $\mathcal{H}$ be a DAG of $N$ nodes representing a valid causal
state sequence, and let $\pi$ be any topological permutation
dictating memory dependencies.  A Bene\v{s} network of depth
$2\log_2(N) - 1$ is rearrangeably non-blocking~\cite{benes}.  By the
Slepian--Duguid theorem, there exists a set of boolean switch
assignments $S \in \{0,1\}^{W \times D}$ that simultaneously routes
all required historical dependencies without edge collisions.
\end{theorem}

\begin{proof}
By induction on the layers of the hypercube, applying Hall's
Marriage Theorem to the bipartite graph of upper and lower
subnetworks.  In our framework, this guarantees that the
prover's deterministic plaintext Waksman pathfinding~\cite{waksman} will never fail
to find a valid routing witness in $\mathcal{O}(N \log N)$ time,
ensuring perfect completeness for any arbitrary DAG reduction.
\end{proof}

\begin{lemma}[Routing Gate Correctness]\label{lem:routing}
Let $s \in \mathbb{F}_2$ be a switch flag with inputs
$a,b \in \mathbb{F}_2^w$ and output $y \in \mathbb{F}_2^w$.
Define the constraint
\begin{equation}\label{eq:routing}
  C_{\mathrm{route}}(s,a,b,y) \;:=\;
    y \;\oplus\; a \;\oplus\; s \cdot (a \oplus b).
\end{equation}
Then $C_{\mathrm{route}} = 0$ if and only if $y = a$ when $s = 0$
and $y = b$ when $s = 1$.
\end{lemma}

\begin{proof}
Because we operate in characteristic~$2$, subtraction coincides
with addition ($\oplus$) and $-1 = 1$.  The standard algebraic
multiplexer $(1-s)\cdot a + s\cdot b - y$ reduces to
$y \oplus a \oplus s \cdot (a \oplus b)$.

\emph{Case $s=0$.}  The switch term vanishes:
$C_{\mathrm{route}} = y \oplus a = 0$, so $y = a$.

\emph{Case $s=1$.}  We obtain
$C_{\mathrm{route}} = y \oplus a \oplus (a \oplus b)
  = y \oplus b = 0$,
so $y = b$.

Since $s$ is constrained to $\mathbb{F}_2$ by the switch-validity
gate $s \cdot (s \oplus 1) = 0$, these are the only two cases.
\end{proof}

\begin{theorem}[LTC Soundness]\label{thm:soundness}
Let $G$ be an $(n, d, \lambda)$-Expander used as the LTC commitment
matrix with relative distance $\delta \ge 0.10$.  If a dishonest
prover submits a trace that differs from every valid trace in at
least one bit, and the verifier draws $k$ column queries uniformly
at random, then
\[
  \Pr[\text{forgery accepted}] \;\le\; (1 - \delta)^{k}.
\]
In particular, setting $k = 843$ yields soundness error
$\le 2^{-128}$.
\end{theorem}

\begin{proof}
By the Expander mixing lemma, the LTC matrix~$G$ amplifies any
nonzero input perturbation so that at least a $\delta$-fraction of
the $W_{\mathrm{stretched}}$ output columns become corrupted.  Each
of the verifier's $k$ queries is drawn independently and uniformly.
The probability that all $k$ queries miss every corrupted column is
at most $(1 - \delta)^k$.  Solving
\[
  (1 - 0.10)^k \;\le\; 2^{-128}
  \quad\Longrightarrow\quad
  k \;\ge\; \frac{128 \ln 2}{\ln(10/9)}
    \;\approx\; 842.1.
\]
Setting $k = 843$ therefore suffices.
\end{proof}

\begin{theorem}[Structural Identity Binding]\label{thm:identity}
Let the arithmetic circuit define a continuous wire geometry where
the output registers of the signature-verification and
content-hashing gadgets map strictly to the input domain of the
Bene\v{s} routing network.  An equivocation forgery is
mathematically impossible without violating the constraint
polynomials of the routing network.
\end{theorem}

\begin{proof}
In a unified execution trace, data is immutable once committed.  If
the payload bytes successfully satisfy the cryptographic identity
constraints, those exact bits inhabit the specific columns routed to
the logic layer.  Attempting to substitute a different bit value
$y \neq a$ at the routing boundary violates the Bene\v{s} spatial
constraint $C_{\mathrm{route}} = 0$, guaranteeing the proof will
fail sum-check verification.
\end{proof}

\section{Case Study: Matrix Protocol State Resolution}\label{sec:matrix}

To demonstrate the practical efficacy of the unified framework, we
apply it to a notoriously demanding DAG-consistency problem: Matrix
Protocol State Resolution~(v2)~\cite{matrix-spec}.  When a
homeserver joins a federated room with tens of thousands of
historical events, standard procedure requires downloading the
entire causal history.  The $\mathcal{O}(N^2)$ worst-case density of
deeply intertwined auth-chains forces joining servers to redundantly
re-perform massive state reductions over untrusted data.

By instantiating our framework with a logic layer configured for
Matrix authorization rules (power-level monotonicity and
lexicographical tie-breaking), we mathematically lift the
$\mathcal{O}(N^2)$ algorithmic burden out of the network consensus
mechanism.  Servers partition the state DAG across the network via
recursive MapReduce proving~(\Cref{sec:recursive}), aggregating the
topological sub-graphs into a singular STARK proof.

Because Matrix is migrating to PQC standards, we evaluate the
constraint profile of baking identity directly into the trace.  For
a state fork comprising 43{,}000 events, proving Canonical JSON
structure and executing native Keccak-256 hashes inside
$\mathbb{F}_2$ requires $\approx 860$ million trace constraints.
Executing 43{,}000 lattice-based FN-DSA signatures requires
$\approx 215$ billion constraints.  While processing 216 billion
binary constraints requires substantial computation---typically 10
to 45 minutes on a heavily parallelized multi-core prover---this
temporal tradeoff is precisely aligned with the architecture of
background epoch rollups.  The proving cost is incurred exactly once
asynchronously, while the entire Matrix network benefits from
sub-centisecond, cryptographically sealed verification.

\subsection{Empirical Benchmark: \texttt{\#community:matrix.org}}\label{sec:bench}

To validate the practical efficiency of the framework, we executed the
full witness generation pipeline against the state snapshot of
\texttt{\#community:matrix.org}, a heavily trafficked public room
containing 43{,}543 state events.  The pipeline performed a

$\S11.20.2$ tie-breaking), resolved the resulting state,
computed the Waksman routing permutation over $N = 65{,}536$ padded
width, and materialized the $\mathbb{F}_2$ execution trace.

\section{Conclusion}\label{sec:conclusion}

We have demonstrated that domain-specific graph arithmetization
radically outperforms general-purpose zkVMs for verifiable DAG
computations.  By replacing CPU-emulated memory arrays with static
Bene\v{s} routing networks, we collapse execution overhead to
strictly $\mathcal{O}(N \log N)$ spatial constraints.  Furthermore,
by leveraging the $\mathbb{F}_2$ arithmetic synergy of Post-Quantum
Cryptography, we introduce a unified, single-layer STARK that
structurally binds payload parsing, identity verification, and state
logic into a single unforgeable cryptographic execution.  Coupled
with recursive SNARK architecture for horizontal distributed
processing, the framework provides a comprehensive mathematical
blueprint for the real-time, trustless verification of highly
concurrent decentralized systems.
