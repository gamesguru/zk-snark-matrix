% ── Abstract ──────────────────────────────────────────────────────
\begin{abstract}
We present a graph-native STARK framework
designed for the verifiable reduction of Directed Acyclic Graphs
(DAGs).  Traditional zero-knowledge virtual machines (zkVMs)
emulate Von Neumann architectures, incurring prohibitive
$\mathcal{O}(C \log C)$ memory permutation overheads across
millions of emulated CPU cycles~($C$).  Our framework bypasses
CPU emulation entirely.  We separate the architecture into two
distinct graphs: a \emph{Bene\v{s}/Superconcentrator routing graph}
that physically enforces memory consistency within the execution
trace without global permutation arguments, and an
\emph{Expander-based Locally Testable Code} (LTC) that stretches
the committed trace such that any single falsified bit cascades
into $\ge 10\%$ corruption---detectable by $k = 843$ random column
queries achieving $2^{-128}$ soundness error.  All trace arithmetic
operates in a binary field using pure bitwise XOR, eliminating the
need for prime-field FFTs.  The prover runs in
$\mathcal{O}(N \log N)$ time, the proof payload is 150--250\,KB,
and the verifier confirms soundness in $\approx 5$\,ms.  We
demonstrate the framework's efficacy by applying it to Matrix
Protocol state resolution, transforming a massively concurrent DAG
reduction into a trustless, sub-centisecond verification.
\end{abstract}

\section{Introduction}\label{sec:intro}

The demand for verifiable computation has driven rapid advancements
in Scalable Transparent Arguments of Knowledge (STARKs).  The
prevailing industry trajectory favors general-purpose
zkVMs~\cite{sp1,jolt}, which compile high-level code into RISC-V
instruction traces and prove their correct execution
cycle-by-cycle.

While general-purpose zkVMs offer developer ergonomics, they
introduce a catastrophic computational bottleneck for applications
processing highly structured, large-scale data---such as
deterministically reducing concurrent states in distributed systems
or verifying large DAGs.  The bottleneck stems fundamentally from
the Von Neumann architecture: the emulation of random-access memory
and instruction cycles.

In a zkVM, proving that a specific value was correctly written and
later read from RAM requires a global memory consistency check.
Techniques like LogUp or offline memory checking force the prover to
construct trace columns proportional to the number of CPU cycles,
computing expensive algebraic fractions over large prime fields.
Our approach abandons the virtual machine.  We observe that a
broad class of verifiable computations---UTXO transaction graphs in
blockchains, dependency resolution in build systems, causal event
ordering in distributed ledgers---share a common structure: they
reduce a DAG by propagating values along topological dependencies.
Rather than emulating this walk cycle-by-cycle inside a VM, we embed
it directly into the STARK execution trace as a static,
\emph{Graph-Native} routing network.  We physically route
historical variables to their destination nodes via a rigidly
constrained Bene\v{s} switching network.  By pairing this localized $\mathcal{O}(N \log N)$
geometric routing with Expander-based LTCs operating strictly in
characteristic~2, our framework drops the prover overhead down
to the absolute limit of native CPU bitwise instructions.

\section{The Memory Consistency Bottleneck}\label{sec:bottleneck}

To understand the asymptotic advantage of graph-native proving, we
must formally contrast it with the standard zkVM memory
checking paradigm utilized by systems like
SP1~\cite{sp1} and Jolt/Lasso~\cite{jolt,lasso}.

\subsection{The zkVM Paradigm: LogUp and Grand Products}

General-purpose zkVMs operate by emulating an Instruction Set
Architecture (ISA).  Let $N$ be the number of nodes in a DAG, and
$C$ be the total CPU cycles required to process it natively.  To
prove RAM consistency, trace-based zkVMs utilize permutation
arguments like LogUp.  Every memory read and write generates a
fractional term $\frac{1}{X - \mathrm{val}}$.  For a standard
topological sort, manipulating pointers and arrays in RISC-V
inflates $C$ to orders of magnitude larger than~$N$ (i.e.,
$C = K \cdot N$, where $K$ is often $10^3$ to~$10^4$).  The prover
is bound by $\mathcal{O}(C \log C)$ Fast Fourier Transforms (FFTs)
over a massive prime field to evaluate these rational functions.

Sum-check-based zkVMs (e.g., Jolt~\cite{jolt}) replace the FFT with
multi-linear extensions and lookup tables (Lasso~\cite{lasso}).
While this achieves strictly $\mathcal{O}(C)$ prover time, it still
inherently tracks the Von Neumann cycle counter, program counter,
and register state.  The constant is overwhelmingly dominated by
instruction decoding and ISA emulation, leaving runtime bounds
chained to $K \cdot N$.

\subsection{Graph-Bound Routing}

We observe that verifying a DAG reduction
does not require an ISA\@.  By projecting topological dependencies
directly onto a static Bene\v{s} permutation network,
our framework replaces the temporal sequence of CPU
cycles~($C$) with a purely spatial 2D trace.  Rather than using
algebraic global permutations to prove that node~$A$ read from
node~$B$, we embed a routing graph directly into
the STARK execution trace.  Variables route through the trace
geometrically via local switch constraints.  Consequently, the
prover time scales strictly with the input size
$\mathcal{O}(N \log N)$, operating entirely via bitwise XORs over
$\mathbb{F}_2$ without a single emulated CPU cycle.

\begin{remark}[Asymptotic Crossover]\label{rem:crossover}
Constant-degree superconcentrator-based Expander graphs~\cite{valiant} achieve a
theoretical $\mathcal{O}(N)$ routing cost, but the hidden constant
is enormous: explicit constructions (e.g., zig-zag products) carry
combinatorial overhead so large that the crossover with
$\mathcal{O}(N \log_2 N)$ Bene\v{s} routing~\cite{benes} occurs only at input
sizes on the order of $N \gtrsim 2^{75}$---far beyond any
foreseeable application.  We therefore adopt the
$\mathcal{O}(N \log N)$ Bene\v{s} network as the practical
construction throughout this work.

Conversely, while Jolt's sum-check protocol achieves strictly
$\mathcal{O}(C)$ cryptographic cost, the per-node ISA emulation
constant $K$ (typically $10^3$--$10^4$) ensures that
$K \cdot N \gg N \log_2 N$ for all practical DAG sizes.  Even at
$N = 2^{20}$ (${\approx}\,10^6$ nodes), the Jolt prover processes
${\sim}\,10^{9}$--$10^{10}$ emulated cycles while our framework
processes ${\sim}\,2 \times 10^{7}$ routing operations---a gap of
two to three orders of magnitude.
\end{remark}

\begin{remark}[Implementation Field]\label{rem:field}
The current proof-of-concept uses the BabyBear prime field
($p = 2^{31}-1$) via Plonky3~\cite{plonky3}, because mature
binary-field STARK tooling (e.g., Binius) was not yet available at
development time.  Migrating to $\mathbb{F}_2$ is a planned
engineering step that strictly improves constant-factor performance
without altering the topological bounds.  The base-field operations
use XOR ($\oplus$) and AND ($\cdot$); extension-field operations
required for Fiat--Shamir challenges in
$\mathbb{F}_{2^{128}}$~\cite{fiat-shamir} use hardware-accelerated
carry-less multiplication (\texttt{CLMUL}).
\end{remark}

\section{Framework Architecture}\label{sec:arch}

The framework is composed of two mathematically isolated graphs: a
dynamically configured internal \emph{Routing Graph} to replace CPU
memory, and a static external \emph{Cryptographic Graph} to replace
polynomial commitment schemes (PCS).

\subsection{Trace Embedding and Bene\v{s} Routing}

The framework initialization separates the DAG computation into a
\emph{public instance} (the topological boundary hashes) and a
\emph{private witness} (the node data, the topological sort
permutation, and pre-computed Waksman routing hints).  Rather than
allocating dynamic memory, the prover embeds this witness into a
static $\mathbb{F}_2$ execution trace of width~$W$ (padded to
$2^d \ge N$) and depth~$D$.  The input layer ingests the initial
DAG state.  Guided by the private routing hints, the intermediate layers
function as a physical Bene\v{s} network, recursively copying
required state variables downward and across adjacent column edges
using native bitwise XOR operations.  A Bene\v{s} network for $N$
inputs consists of $2\log_2 N - 1$ layers of $N/2$ switches each,
totalling $(2\log_2 N - 1) \cdot N/2 = \mathcal{O}(N \log N)$
routing constraints.  Each switch enforces a single local algebraic
constraint over $\mathbb{F}_2$:
\begin{equation}\label{eq:switch}
  \texttt{out} = a \oplus s \cdot (a \oplus b),
\end{equation}
where $s \in \{0,1\}$ is the private switch state.
This purely spatial
construction guarantees that all data dependencies arrive at the
final logic layer for local constraint evaluation without invoking a
global memory bus.

\subsection{Expander-Based Commitment (LTC)}

Standard STARKs commit to the execution trace by interpolating it
into polynomials and generating Reed--Solomon codewords via FFTs.
Our framework replaces this algebraic commitment with proximity
testing over a sparse Cryptographic Graph.  The prover stretches the
binary grid by multiplying it against a public, sparse,
constant-degree Expander matrix~$G$:
\[
  T_{\mathrm{stretched}} \;=\; T_{\mathrm{original}} \cdot G
\]
Because $G$ is a highly sparse, constant-degree matrix operating
over $\mathbb{F}_2$, each output element requires only a fixed
number of XOR operations.  For a Bene\v{s} trace with
$\mathcal{O}(N \log N)$ nodes, the total cost is
$\mathcal{O}(N \log N)$ bitwise XORs.  By the Expander mixing
lemma, any localized constraint failure---such as a single falsified
Bene\v{s} switch flag---cascades to corrupt a $\ge \delta$-fraction
(e.g., $10\%$) of the extended grid.  The prover then Merkle-commits
these stretched columns to produce the trace commitment root.

\subsection{Non-Interactive Verification}

Verification is orchestrated via a Fiat--Shamir hash
sponge~\cite{fiat-shamir}, requiring strictly $\mathcal{O}(1)$
cryptographic interactions and $\mathcal{O}(\log N)$ constraint
evaluations.  The verifier seeds the sponge with the public instance
to derive challenge indices.  The prover collapses the 2D routing
and logic constraints into a condensed 1D sum-check summary, which
the verifier confirms evaluates to zero.  Finally, the verifier
authenticates $k$ randomly drawn columns from the Merkle tree and
performs local LTC proximity checks by multiplying the un-stretched
segments by~$G$.  Because a forgery corrupts a $\delta$-fraction of
the trace, drawing $k$ independent columns bounds the soundness
error strictly to $(1 - \delta)^k$.

\section{Theoretical Foundations}\label{sec:theory}

\begin{theorem}[Non-Blocking Trace Embeddability]\label{thm:embed}
Let $\mathcal{H}$ be a DAG of $N$ nodes representing a valid causal
state sequence, and let $\pi$ be any topological permutation
dictating memory dependencies.  A Bene\v{s} network of depth
$2\log_2(N) - 1$ is rearrangeably non-blocking~\cite{benes}.  By the
Slepian--Duguid theorem, there exists a set of boolean switch
assignments $S \in \{0,1\}^{W \times D}$ that simultaneously routes
all required historical dependencies without edge collisions.
\end{theorem}

\begin{proof}
By induction on the layers of the hypercube, applying Hall's
Marriage Theorem to the bipartite graph of upper and lower
subnetworks.  In our framework, this guarantees that the
prover's deterministic plaintext Waksman pathfinding~\cite{waksman} will never fail
to find a valid routing witness in $\mathcal{O}(N \log N)$ time,
ensuring perfect completeness for any arbitrary DAG reduction.
\end{proof}

\begin{lemma}[Routing Gate Correctness]\label{lem:routing}
Let $s \in \mathbb{F}_2$ be a switch flag with inputs
$a,b \in \mathbb{F}_2^w$ and output $y \in \mathbb{F}_2^w$.
Define the constraint
\begin{equation}\label{eq:routing}
  C_{\mathrm{route}}(s,a,b,y) \;:=\;
    y \;\oplus\; a \;\oplus\; s \cdot (a \oplus b).
\end{equation}
Then $C_{\mathrm{route}} = 0$ if and only if $y = a$ when $s = 0$
and $y = b$ when $s = 1$.
\end{lemma}

\begin{proof}
Because we operate in characteristic~$2$, subtraction coincides
with addition ($\oplus$) and $-1 = 1$.  The standard algebraic
multiplexer $(1-s)\cdot a + s\cdot b - y$ reduces to
$y \oplus a \oplus s \cdot (a \oplus b)$.

\emph{Case $s=0$.}  The switch term vanishes:
$C_{\mathrm{route}} = y \oplus a = 0$, so $y = a$.

\emph{Case $s=1$.}  We obtain
$C_{\mathrm{route}} = y \oplus a \oplus (a \oplus b)
  = y \oplus b = 0$,
so $y = b$.

Since $s$ is constrained to $\mathbb{F}_2$ by the switch-validity
gate $s \cdot (s \oplus 1) = 0$, these are the only two cases.
\end{proof}

\begin{theorem}[LTC Soundness]\label{thm:soundness}
Let $G$ be an $(n, d, \lambda)$-Expander used as the LTC commitment
matrix with relative distance $\delta \ge 0.10$.  If a dishonest
prover submits a trace that differs from every valid trace in at
least one bit, and the verifier draws $k$ column queries uniformly
at random, then
\[
  \Pr[\text{forgery accepted}] \;\le\; (1 - \delta)^{k}.
\]
In particular, setting $k = 843$ yields soundness error
$\le 2^{-128}$.
\end{theorem}

\begin{proof}
By the Expander mixing lemma, the LTC matrix~$G$ amplifies any
nonzero input perturbation so that at least a $\delta$-fraction of
the $W_{\mathrm{stretched}}$ output columns become corrupted.  Each
of the verifier's $k$ queries is drawn independently and uniformly.
The probability that all $k$ queries miss every corrupted column is
at most $(1 - \delta)^k$.  Solving
\[
  (1 - 0.10)^k \;\le\; 2^{-128}
  \quad\Longrightarrow\quad
  k \;\ge\; \frac{128 \ln 2}{\ln(10/9)}
    \;\approx\; 842.1.
\]
Setting $k = 843$ therefore suffices.
\end{proof}

\section{Case Study: Matrix Protocol State Resolution}\label{sec:matrix}

To demonstrate the efficacy of the proposed framework, we
apply it to a notoriously demanding DAG-consistency problem: Matrix
Protocol State Resolution~(v2)~\cite{matrix-spec}.

\subsection{Matrix State Resolution v2}

A Matrix room's history forms a DAG\@.  When a homeserver
\emph{joins} a federated room it must obtain and verify the room's
\emph{state}---the cumulative result of applying every authorization
event from genesis through a deterministic State Resolution
algorithm.  Two strategies exist today: (i)~\textbf{Full Join}---download
the entire Auth Chain and re-execute locally, which is
cryptographically sound but prohibitively expensive for large rooms;
(ii)~\textbf{Partial Join}~(MSC\,3902)---accept the resident
server's claimed state on trust, which is fast but a significant
compromise on decentralization.

By instantiating our framework with a logic layer configured
for Matrix authorization rules (power-level monotonicity and
lexicographical tie-breaking), we entirely eliminate the need for
local full-DAG re-execution.  We propose
\texttt{GET\allowbreak\ /\_matrix/\allowbreak federation/\allowbreak v1/\allowbreak zk\_state\_proof/\allowbreak\{roomId\}}
as the federation endpoint.
Proofs are generated asynchronously as periodic \emph{epoch rollups}
(e.g., bi-weekly), avoiding latency on the join path.

\subsection{Identity vs.\ Logic Separation}\label{sec:identity}

A critical design constraint is the strict separation of cryptographic
identity from protocol logic.  A na\"ive zkVM implementation would
attempt to verify Ed25519 signatures inside the AIR trace,
triggering catastrophic non-native arithmetic blowup: Ed25519
operates over a 255-bit prime field ($p = 2^{255} - 19$), while
the STARK trace operates entirely in $\mathbb{F}_2$.  Arithmetizing
a single scalar multiplication would require $\sim$500{,}000
constraints, dwarfing the $\sim$1{,}000{,}000 constraints needed for
the entire 43{,}543-event routing network.

Our framework delegates signature verification to the
\textbf{prover's native Host OS}, which authenticates \emph{all}
events in the historical auth chain using optimized CPU instructions.
The prover then computes $h_{\mathrm{auth}} =
\text{Keccak-256}(\texttt{event\_id}_1 \| \sigma_1 \| \cdots
\| \texttt{event\_id}_n \| \sigma_n)$ over the verified events
and commits it as a public input in the Fiat--Shamir transcript.
The STARK proof mathematically guarantees: \emph{``state resolution
was executed correctly on a DAG of events that had valid signatures.''}

The \textbf{verifier} (joining guest server) does \emph{not} receive
the auth chain---that is the entire value proposition.
The verifier receives only the current state events and the proof.
It checks Ed25519 signatures on \emph{all} received state events
($\mathcal{O}(N)$ native verification as a DoS short-circuit), then validates
the proof's \texttt{da\_root} against the federation via multi-server
attestation (\Cref{sec:bench}).  If the honest minority of federation
peers agrees on the same \texttt{da\_root}, the hidden auth chain
is confirmed complete without the verifier ever downloading it.

\subsection{Proof Payload}

The proof is transmitted over the Matrix Federation API as:

\begin{lstlisting}[frame=single,basicstyle=\ttfamily\footnotesize]
{
  "public_journal": {
    "da_root":    "0xc29ab9db...",
    "state_root": "0x259df515...",
    "h_auth":     "0x...",
    "n_events":   43543,
    "n_resolved": 43543
  },
  "zk_proof_bytes": {
    "trace_commitment_root": "0xROOT...",
    "sum_check_summary": ["0x..."],
    "queried_columns_data": [[...], [...]],
    "merkle_auth_paths": [[...], [...]]
  }
}
\end{lstlisting}

\noindent\textbf{Total payload size: 150--250\,KB}
(dominated by $k = 843$ Merkle authentication paths at tree depth
$\approx 17$; aggressive path batching reduces this further).
For a 10,000-event fork, the prover generates the proof in under a
second; the verifier confirms it in $\approx 5$\,ms.

\paragraph{Topological AIR (\texttt{ruma-zk-topological-air}).}
Defines the trace layout, the \texttt{MatrixEvent} data type, the
topological constraint function, and the canonical verification key
hash.  The PoC operates over BabyBear via Plonky3; the production
target is $\mathbb{F}_2$ (see \Cref{rem:field}).
The crate is \texttt{\#![no\_std]}.

\paragraph{Prover (\texttt{ruma-zk-prover}).}
Orchestrates Phase 1 (Kahn sort), Phase 2 (Waksman pathfinding and
trace embedding), and Phase 3 (LTC stretch and Merkle commit).
The internal topology is a Boolean Hypercube~$H_d$ with Bene\v{s}
routing (padding to $2^d$ rather than $n!$).

\paragraph{Verifier (\texttt{ruma-zk-verifier}).}
Compiles to three targets: native Rust (\texttt{lib}),
WebAssembly (\texttt{wasm-bindgen}), and
Android/iOS (Mozilla UniFFI, \texttt{cdylib}).
Both prover and verifier enforce
\texttt{\#![forbid(unsafe\_code)]}.

\medskip

Machine-checked proofs accompany the Rust implementation, organized
in the \texttt{lean\_src/} directory of the topological-AIR crate (Mathlib).
\texttt{Arithmetization.lean} formalizes the asymptotic advantage of
topological routing over Von Neumann permutation arguments: using an
Expander Superconcentrator the topological coprocessor achieves
strict $\mathcal{O}(N)$ cost.  (The Bene\v{s} network used in the
current PoC achieves $\mathcal{O}(N \log N)$, still without a
prime-field permutation argument.)
\texttt{Commitment.lean} proves that the topological GKR approach
isolates PCS overhead to the input layer and formally verifies the
Expander expansion bounds utilized in \Cref{thm:soundness}.
\texttt{StarGraphEmbedding.lean} axiomatizes that for any
Kahn-sorted event list, a valid routing-graph walk embedding exists;
extending to the Hypercube/Bene\v{s} network is planned future work.
The current formalization relies on six \texttt{axiom} declarations
across the codebase: \texttt{exp\_one\_lt\_three} (numerical bound),
\texttt{rejection\_prob\_le\_one} and
\texttt{self\_diagnosing\_expansion} (Expander properties),
\texttt{isStarGraphStep} and
\texttt{exists\_star\_graph\_embedding} (routing existence), and
\texttt{dag\_has\_zero\_in\_degree} and
\texttt{gray\_code\_step} in the auxiliary \texttt{ruma-lean} crate.
Closing these axioms with constructive proofs is an explicit goal.


\subsection{Empirical Benchmark: \texttt{\#community:matrix.org}}\label{sec:bench}

To validate the practical efficiency of the framework, we executed the
full witness generation pipeline against the state snapshot of
\texttt{\#community:matrix.org}, a heavily trafficked public room
containing 43{,}543 state events.  The pipeline performed a
deterministic topological sort (Kahn's algorithm with
$\S11.20.2$ tie-breaking), resolved the resulting state,
computed the Waksman routing permutation over $N = 65{,}536$ padded
width, and materialized the $\mathbb{F}_2$ execution trace.
All timings are single-threaded release builds on commodity hardware
(Intel i7-8700K, 32\,GB DDR4).

\section{Security Considerations and Future Work}\label{sec:security}

\paragraph{Threat Model.}
A malicious server substituting a trivial ``always accept'' circuit
is mitigated by binding each room version to a canonical verification
key hash; the verifier rejects mismatched keys.  Soundness is
guaranteed to $2^{-128}$ via LTC error amplification and $k = 843$
random samples (\Cref{thm:soundness}).  While a valid proof attests
to correct execution, it does not guarantee data availability; a
malicious server withholding underlying events must be addressed
orthogonally at the protocol layer.  Memory safety is enforced via
\texttt{\#![forbid(unsafe\_code)]} across all crates.

\paragraph{Future Work.}
The current proof-of-concept uses a Star Graph walk as scaffolding;
the production system requires a complete Waksman routing network
over the Boolean Hypercube, together with full AIR binding to a
dedicated binary-field backend (e.g., Binius).  Richer constraint
layers encoding full Matrix auth rules (join rules, redactions,
event-type restrictions) are needed for production coverage.
Recursive composition---wrapping proofs in a Groth16 SNARK for
constant-size verification---and closing the six remaining Lean~4
\texttt{axiom} declarations are ongoing research directions.

\section{Conclusion}\label{sec:conclusion}

We have demonstrated that domain-specific graph
arithmetization can radically outperform general-purpose zkVMs for
verifiable DAG computations.  By replacing CPU-emulated memory
arrays with static Bene\v{s} routing networks, and replacing
prime-field FFTs with Expander-based Locally Testable Codes, we
collapse the cryptographic overhead to its theoretical floor:
$\mathcal{O}(N \log N)$ binary-field proving (with a theoretical
$\mathcal{O}(N)$ limit via Superconcentrators), 150--250\,KB proof
payloads, and ${\approx}\,5$\,ms verification---all with
$2^{-128}$ soundness error.  The framework provides a concrete path
toward trustless, real-time verification of highly concurrent
decentralized systems.
